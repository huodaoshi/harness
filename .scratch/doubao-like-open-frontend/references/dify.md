# 参考：Dify

**类型：** 间接竞品
**链接：** https://github.com/langgenius/dify
**来源：** Agent 搜索
**置信度：** 高
**搜索日期：** 2026-05-19

## 他们解决什么

LLM 应用开发平台：工作流、RAG、Agent、**自带 Web 控制台与对话应用**。

## 目标用户

要搭产品级 AI 应用、需要编排与发布的团队。

## 核心功能（与本次想法相关的）

- 官方 Next.js `web`：工作室 + 已发布 Chat 应用
- 对话 API、知识库、工具、多租户（视部署）
- 社区有轻量「仅 Chat」WebUI 分叉（如 simple-chat-webui-for-dify）

## 定价 / 商业模式

开源可自托管；有云版 SaaS（定价未逐条核实）。

## 优点

- 「聊天」只是表层，后端编排强
- 适合豆包式「助手 + 工具」但由工作流驱动

## 缺点 / 局限

- **不是**专用聊天前端库；UI 风格偏 B 端控制台
- 只要前端壳 -> 过重，或需二次封装 Chat 应用

## 与我们的关系

不适用作纯 UI 库 — 若业务含工作流/RAG 后台可整体采用。
