## 要构建什么

新建 **Web 客户端**（PWA 友好）：首屏免责声明勾选 + 主按钮「我现在很难受」进入 **distress** 模式聊天；聊天页用 `POST /v1/sessions/stream` 消费 SSE，增量渲染 assistant 气泡；展示当前会话消息列表；网络/服务端 `error` 事件可见提示。

本阶段可对接 #01–#03 后端（假模型即可）。**危机卡片 UI** 在 #02 完成后需能展示 `crisis` 事件并禁用输入（若 #02 未合并，本 issue 内用 stub 事件自测 UI）。

## 验收标准

- [ ] 未勾选免责声明不可进入聊天
- [ ] 点击洪峰按钮后以 `mode=distress` 发起流式会话
- [ ] 页面可见流式打字效果直至 `done`
- [ ] 收到 `crisis` 时展示危机卡片且不可继续发送
- [ ] 移动端浏览器布局可用（单列、输入框不被键盘永久遮挡）
- [ ] 可跳过档案直接进入聊天（#5、#6 未完成时不阻塞）
