## 执行摘要

为关注**原生家庭与亲密关系**、需要情绪宣泄与自我认知、但尚未或不愿进入重度临床咨询的个体，提供垂直于「关系情绪」的 **AI 倾诉伙伴**——体验上对齐豆包式对话流畅度，产品上聚焦**情绪洪峰倾诉、可编辑关系档案、安全与危机护栏**，明确非医疗、非玄学、非虚拟恋爱。

建议 **8 周内交付 Web 优先 MVP**（C1+C2 最小+C3），通过红队、危机剧本与隐私评审后再进入 v1.1（档案强化、轻复盘、App 上架）。**商业模式**与**记忆策略（是否接近豆包式泛记忆 / 向量 RAG）**尚未在本场议会定案，须单开会议；**技术方案**按本场共识写入下文，但**须后续专项重审**（用户明确要求）。

**是否建议立项：** 建议以「验证楔子」立项——在 30 人内测与门禁通过前，不投入工具库、社区、付费与全文向量记忆。

## 问题与机会

### 用户/业务问题

- 用户在**家庭/伴侣冲突或情绪洪峰**时，通用 AI（如豆包）「说得对但接不住具体关系脉络」。
- 测测等产品的关系场景常被套在**玄学/测试/社区**框架内；林间等偏**泛情绪成长**，家庭/关系非第一叙事。
- 用户要的是**倾诉与被理解**，多数尚未准备好付费真人咨询（羞耻、价格、时间）。

### 为何现在做（证据或假设）

- 大模型 API 与国内云审成熟，**小团队可编排垂直会话**（未经本场技术专项确认，见下文「技术方案状态」）。
- 品类已有付费习惯（泛心理 App），但 **AI+关系垂直** 的清晰产品位仍不充分（置信度：中，待访谈验证）。

## 目标用户与核心场景

### 画像（Primary Persona）

**小雨，32 岁，一线城市白领。** 与父母边界不清、伴侣沟通反复踩雷；用过豆包但觉得接不住；可能玩过测测测试；**不愿**立刻找咨询师。

### 核心场景

1. **情绪洪峰（当晚）：** 吵架/被催婚/翻旧账后，15–30 分钟倾诉与情绪命名。  
2. **关系复盘（周末，v1.1）：** 理清模式与边界（非诊断）。  
3. **日常微支持（可选）：** 短 check-in，防积压成洪峰。

## 产品形态与 MVP

### 推荐形态

- **对外：** 「关系情绪 AI」/「AI 倾诉」——介于 AI 挚友与咨询助理之间。  
- **对内楔子：** 情绪洪峰倾诉。  
- **客户端：** Web PWA **优先**；App（RN/uni-app）可与 MVP 并行，**不阻塞** W3 验证。

### MVP 包含（C1–C3）

| 能力 | 说明 |
| ---- | ---- |
| **C1 关系情绪洪峰倾诉** | 一键「我现在很难受」；倾听为主；单次会话质量可评 |
| **C2 关系档案（用户主导）** | 用户可编辑：我 / 重要他人 / 当前议题；可选跳过；近 1 次会话摘要（MVP 最小） |
| **C3 安全与边界** | 免责声明；自伤/家暴/医疗三类危机固定流程；禁止诊断与用药建议 |

**可选半项：** 会话结束 3 句小结卡（非完整复盘模块）。

### MVP 不包含

工具库、社区、测试/玄学、虚拟恋爱、真人撮合、语音实时通话、付费体系、向量全文 RAG（记忆 Level 4）、自研基座模型。

### 成功指标（内测，谨慎解读）

- 盲测：≥60% 认为比豆包（关系向 prompt）「更贴关系」  
- 7 日留存：正向信号（n≥30，不作硬性对外承诺）  
- 用户原话：「愿下周再来」/ 愿推荐

## 技术选型

> **状态：草案 — 须后续「技术方案」专项会议重审。** 下文为 product-council 第 4–5 轮共识，用户已锁定 Hertz/Eino 方向，但整体实现细节、中间件、部署与模型供应商仍以专项为准。

### 推荐（议会草案，待重审）

| 层 | 选型 | 理由 |
| ---- | ---- | ---- |
| HTTP | **Hertz**（CloudWeGo） | 与用户指定的 Eino 同生态 |
| 编排 | **Eino**（Go）Graph/Workflow | 会话状态机、Aspect（安全/脱敏）、流式 |
| 存储 | PostgreSQL（档案、摘要）+ Redis（窗口、限流） | 结构化记忆，避免 MVP 全量向量 |
| 模型 | 火山方舟 / 豆包等国内 API | 中文与延迟；`ChatModel` 接口预留 failover |
| 安全 | 规则 SafetyGate + 云内容安全 API | 危机 Branch 不调 LLM |
| 托管 | 国内云单区域 | 数据出境与审核风险 |
| 客户端 | Web 优先；RN/uni-app 可选 | 验证楔子不依赖 App 上架 |

### 架构草图（逻辑层，待重审）

```text
Client (Web/App)
    → Hertz（鉴权、限流、SSE）
    → Eino RelationshipSessionGraph
         Input → SafetyGate → ModeRouter(洪峰|普聊|危机)
         → ProfileInject(PG) → ChatModel(API) → PostProcess(摘要)
    → PostgreSQL / Redis
```

**扩展口（v2）：** ReferralTool（外链转介）；可选 Retriever（摘要级 RAG，非全文）；第二 LLM Provider。

### 备选（何时切换 — 待技术专项确认）

| 条件 | 方向 |
| ---- | ---- |
| 仅小程序可上线 | 微信云开发 + 同一 API（牺牲推送与体验） |
| 红队穿透率高 | 加强规则层或 moderation 供应商 |
| 记忆策略会议决定 Level 3–4 | 增加向量库与评测体系（非 MVP） |

### 待验证（≤1 周 spike）

- [ ] Hertz + Eino 假模型流式端到端  
- [ ] 50 条红队 + 危机 10 剧本  
- [ ] 20 条关系洪峰对话延迟与穿透过率  
- [ ] **技术专项：** 栈版本锁定、目录结构、是否与 CloudWeGo 其余组件（如 Kitex）接缝

## 记忆策略（待议 — 未成文定案）

议会倾向 **MVP 采用 Level 1–2**（用户编辑档案 + 近几次会话摘要），**不默认** Level 4 向量「全记忆」RAG。是否与豆包式「泛记忆」对齐、是否引入 Level 3 摘要向量检索，**须单开「记忆策略」会议**，本场不写入实施承诺。

| Level | 内容 | MVP 倾向 |
| ----- | ---- | -------- |
| 0 | 仅当前会话 | — |
| 1 | 用户编辑关系档案 | ✓ 主档 |
| 2 | 每场结束短摘要，按时间取最近 N 场 | ✓ 最小（N=1 起） |
| 3 | 仅对摘要做向量检索 | 待议 |
| 4 | 全文分块向量 RAG | 默认不做 |

## 范围外（摘要）

**永不：** 诊断/用药建议、玄学主路径、虚拟恋爱、UGC 社区、达人连麦平台、全自动病因判定、运营可读全文永久存档、自研基座模型。

**三年内不做（除非验证推翻）：** 200+ 工具库、实时语音通话、角色/IP 社区、全文向量 RAG、微信小程序为主战场、B2B EAP、与咨询平台 API 深度集成、**付费体系（待商业会议）**。

**触发重议：** 豆包/微信官方关系智能体碾压；监管要求资质；重大危机漏判；留存连续低迷。

---

## 风险与未消解质疑

1. **H3：** 用户是否愿离开豆包——最大市场风险。  
2. **记忆记错：** 若提升自动记忆而未做评测，关系场景信任崩塌。  
3. **技术草案未专项评审：** Hertz/Eino 版本、团队技能、8 周工期可能乐观。  
4. **非临床边界：** 文案与危机 SOP 任一失败均可导致下架或舆情。  
5. **竞品窗口：** 壹心理/豆包 AI 能力进展（references 多项待核实）。  
6. **无商业模式：** v1.1 后可持续性未定义（已知搁置）。
