## 执行摘要

为关注**原生家庭与亲密关系**、需要情绪宣泄与自我认知、但尚未或不愿进入重度临床咨询的个体，提供垂直于「关系情绪」的 **AI 倾诉伙伴**——体验上对齐豆包式对话流畅度，产品上聚焦**情绪洪峰倾诉、可编辑关系档案、安全与危机护栏**，明确非医疗、非玄学、非虚拟恋爱。

建议 **8 周内交付 Web 优先 MVP**（C1+C2 最小+C3），通过红队、危机剧本与隐私评审后再进入 v1.1（档案强化、轻复盘、App 上架）。**技术栈**经 4 轮技术专项拍板：**Go + Hertz + Eino + MongoDB（自建）+ Redis + 国内 LLM API**；**商业模式**与**记忆策略 Level 3–4** 仍须单开会议。

**是否建议立项：** 建议以「验证楔子」立项——在 30 人内测与门禁通过前，不投入工具库、社区、付费与全文向量记忆。

## 问题与机会

### 用户/业务问题

- 用户在**家庭/伴侣冲突或情绪洪峰**时，通用 AI（如豆包）「说得对但接不住具体关系脉络」。
- 测测等产品的关系场景常被套在**玄学/测试/社区**框架内；林间等偏**泛情绪成长**，家庭/关系非第一叙事。
- 用户要的是**倾诉与被理解**，多数尚未准备好付费真人咨询（羞耻、价格、时间）。

### 为何现在做（证据或假设）

- 大模型 API 与国内云审成熟，**小团队可编排垂直会话**（技术专项已确认 Hertz+Eino+自建 Mongo 路径，W1 spike 待执行）。
- 品类已有付费习惯（泛心理 App），但 **AI+关系垂直** 的清晰产品位仍不充分（置信度：中，待访谈验证）。

## 目标用户与核心场景

### 画像（Primary Persona）

**小雨，32 岁，一线城市白领。** 与父母边界不清、伴侣沟通反复踩雷；用过豆包但觉得接不住；可能玩过测测测试；**不愿**立刻找咨询师。

### 核心场景

1. **情绪洪峰（当晚）：** 吵架/被催婚/翻旧账后，15–30 分钟倾诉与情绪命名。  
2. **关系复盘（周末，v1.1）：** 理清模式与边界（非诊断）。  
3. **日常微支持（可选）：** 短 check-in，防积压成洪峰。

## 产品形态与 MVP

### 推荐形态

- **对外：** 「关系情绪 AI」/「AI 倾诉」——介于 AI 挚友与咨询助理之间。  
- **对内楔子：** 情绪洪峰倾诉。  
- **客户端：** Web PWA **优先**；App（RN/uni-app）可与 MVP 并行，**不阻塞** W3 验证。

### MVP 包含（C1–C3）

| 能力 | 说明 |
| ---- | ---- |
| **C1 关系情绪洪峰倾诉** | 一键「我现在很难受」；倾听为主；单次会话质量可评 |
| **C2 关系档案（用户主导）** | 用户可编辑：我 / 重要他人 / 当前议题；可选跳过；近 1 次会话摘要（MVP 最小） |
| **C3 安全与边界** | 免责声明；自伤/家暴/医疗三类危机固定流程；禁止诊断与用药建议 |

**可选半项：** 会话结束 3 句小结卡（非完整复盘模块）。

### MVP 不包含

工具库、社区、测试/玄学、虚拟恋爱、真人撮合、语音实时通话、付费体系、向量全文 RAG（记忆 Level 4）、自研基座模型。

### 成功指标（内测，谨慎解读）

- 盲测：≥60% 认为比豆包（关系向 prompt）「更贴关系」  
- 7 日留存：正向信号（n≥30，不作硬性对外承诺）  
- 用户原话：「愿下周再来」/ 愿推荐

## 技术选型

> **状态：技术专项已评审（2026-05-19，4 轮）。** 模型供应商具体合同、云审厂商名、Eino 精确版本号在 W1 spike 后锁定。记忆 Level 3–4、商业模式仍待议。

### 已拍板栈

| 层 | 选型 | 理由 |
| ---- | ---- | ---- |
| HTTP | **Hertz**（CloudWeGo） | 与 Eino 同生态；SSE、鉴权、限流 |
| 编排 | **Eino**（Go）Graph/Workflow | 分支编排、流式、Aspect（脱敏） |
| 主库 | **MongoDB（自建）** | 关系档案/会话/摘要文档模型；用户明确要求自建 |
| 缓存 | **Redis** | 滑动窗口、限流、热 session |
| 模型 | 火山方舟 / 豆包等国内 API | 中文与延迟；`ChatModel` 接口预留 failover |
| 安全 | **SafetyGate**（L1 规则 + L2 云内容安全） | 危机硬分支不调 LLM；上线前云审 fail-closed |
| 托管 | 国内云单区域；**Mongo 与应用同 VPC 自建** | 数据不出境；运维自担备份与监控 |
| 客户端 | Web PWA 优先 | 验证楔子不依赖 App 上架 |

**MVP 明确不做：** Kitex、向量库、PG 双写、自研基座模型。

### 架构草图（逻辑层）

```text
Client (Web)
    → Hertz: POST /v1/sessions/stream (SSE)
    → Eino RelationshipSessionGraph
         Input → SafetyGate(L1规则→L2云审)
              → crisis → CrisisBranch(固定文案, 无 LLM)
              → block  → END
              → pass   → ModeRouter(洪峰|普聊)
              → ProfileInject(Mongo: profile + lastSummary)
              → ChatModel(stream) → PostProcess(summary3 → Mongo)
    → MongoDB（自建）/ Redis
```

**SSE 事件（最小）：** `token` | `done` | `crisis` | `error`

### MongoDB 集合（MVP）

| 集合 | 用途 |
| ---- | ---- |
| `relationship_profiles` | 用户编辑档案：self、people[]、currentIssue |
| `sessions` | 会话元数据、mode、gateResult；消息可嵌套或分集合，单会话轮次硬 cap |
| `session_summaries` | PostProcess 写入的 3 句摘要（Level 2，MVP N=1） |
| `audit_events`（可选） | 危机/门禁事件；不落用户原文 |

**自建运维最小集：** 单节点或副本集（内测可单节点）；`mongodump` 定时备份；磁盘与连接数监控；按 `userId` 级联删除支持隐私评审。

### SafetyGate（C3 实现要点）

| 级别 | 示例 | 动作 | LLM |
| ---- | ---- | ---- | --- |
| `crisis_self_harm` | 自杀、自残表述 | 固定危机包 + 热线/资源 | **否** |
| `crisis_violence` | 家暴、人身威胁 | 固定包 + 求助指引 | **否** |
| `medical_boundary` | 用药、诊断请求 | **MVP 固定模板**（v1.1 可议窄 LLM） | **否** |
| `block` | 色情/仇恨等 | 拒绝 | **否** |
| `pass` | 其余 | → ModeRouter | 是 |

- 规则外置 `safety_rules_v1.yaml`；危机文案 `crisis_templates/zh-CN.json`。
- Graph **编译期** crisis 边不连接 `ChatModel`；测试断言危机路径 **LLM 调用次数 = 0**。
- 日志：危机事件记 gateResult，**不落用户原文**（仅 hash 或截断）。

### 备选（何时切换）

| 条件 | 方向 |
| ---- | ---- |
| 仅小程序可上线 | 微信云开发前端 + **同一 Hertz API** |
| 红队穿透率 >5% | 加强 L1 词表或换云审供应商 |
| 记忆会议决定 Level 3–4 | 增加向量库与评测（非 MVP） |
| 自建 Mongo 运维吃紧 | 迁托管 Mongo（数据迁移脚本） |
| Eino alpha API 频繁破坏 | pin 版本；最坏情况薄编排（纯 Go switch） |

### Spike 清单

**P0 — W1 必须绿（用户确认 W1 = S1–S3）：**

| ID | 内容 | 通过标准 |
| ---- | ---- | -------- |
| S1 | Hertz→Eino→假 ChatModel→SSE | 浏览器连续 `token` |
| S2 | 危机 10 剧本 | 10/10 `crisis`；LLM 调用 = 0 |
| S3 | Mongo 档案 + 最近 1 摘要 inject | 20 条自动化 |

**P1 — W2–W3：**

| ID | 内容 | 通过标准 |
| ---- | ---- | -------- |
| S4 | 红队 50 条 | 穿透 ≤5%；危机 0 漏判 |
| S5 | 真 LLM 20 条洪峰 | P95 首 token 达标（内测自定） |
| S6 | 云审联调 | fail-closed 符合配置 |
| S7 | 删号级联 | userId 下数据清空 |

**P2 — 门禁前（W6–W7）：** S8 隐私评审；S9 医疗边界 15 条。

### 部署（MVP）

| 项 | 选型 |

## 范围外（摘要）

**永不：** 诊断/用药建议、玄学主路径、虚拟恋爱、UGC 社区、达人连麦平台、全自动病因判定、运营可读全文永久存档、自研基座模型。

**三年内不做（除非验证推翻）：** 200+ 工具库、实时语音通话、角色/IP 社区、全文向量 RAG、微信小程序为主战场、B2B EAP、与咨询平台 API 深度集成、**付费体系（待商业会议）**。

**触发重议：** 豆包/微信官方关系智能体碾压；监管要求资质；重大危机漏判；留存连续低迷。

## 风险与未消解质疑

1. **H3：** 用户是否愿离开豆包——最大市场风险。  
2. **记忆记错：** 若提升自动记忆而未做评测，关系场景信任崩塌。  
3. **自建 Mongo 运维：** 备份/监控/升级由团队自担；单节点故障影响内测。  
4. **Eino alpha 与 8 周工期：** 版本 pin 与 spike 未绿前仍可能延期。  
5. **非临床边界：** 文案与危机 SOP 任一失败均可导致下架或舆情。  
6. **竞品窗口：** 壹心理/豆包 AI 能力进展（references 多项待核实）。  
7. **无商业模式：** v1.1 后可持续性未定义（已知搁置）。
