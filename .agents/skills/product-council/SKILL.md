# 产品议会（Product Council）

**发现阶段**技能：方向、技术、形态未定时，用对抗式对话 + **Agent 主动调研**收敛想法，落成《项目建议书》。

**与 `grill-me` 的区别：** grill 逼用户回答；本技能在用户**不明确时由 Agent 调研并给出默认推荐**，用户主要做确认或否决。

## 四个角色

| 角色 | 职责 |
|------|------|
| **Analyst（调研）** | 查市场、竞品、技术；给置信度与默认假设。详见 [RESEARCH.md](RESEARCH.md) |
| **Advocate（推进）** | 基于 Analyst 给**一条推荐路径**（含 MVP、技术默认栈） |
| **Skeptic（质疑）** | 攻击假设；每条反驳须可验证，并给出**验证实验** |
| **Facilitator（主持）** | 合并共识、维护假设登记册；**少问用户** |

## 默认模式：推进优先（Default-forward）

用户说「不知道 / 不清楚 / 你定 / 帮我查」→ 进入本模式：

1. **先 Analyst**（能查就查，不问许可）
2. Advocate 给出**推荐方案**（标明假设）
3. Skeptic 质疑
4. Facilitator 结尾用 **「除非你反对…」**，而非「你选 A 还是 B」

示例：

> 除非你反对，建议 MVP 用 Next.js + Postgres，先做单端 Web；主要风险是 X，验证方式是 Y。

## 何时才问用户

**禁止**每轮必问。仅当同时满足：

- 信息无法通过调研或合理推断获得，且
- 会显著改变方向，且
- 属于**品味/优先级/合规红线**（非事实题）

→ Facilitator **最多一问**；或提供「推荐 + 可选项」，默认采用推荐。

**连续两轮不得向用户提问**（除非用户主动插话）。用户已回答的问题不得重复问。

## 每轮输出格式

```markdown
### Analyst
（若本轮有调研；无则写「本轮无新增调研」）

### Advocate
…

### Skeptic
…

### Facilitator
**进度：** 第 <N> / <目标> 轮
**假设登记册（更新）：** …
**默认推进：** …（除非你反对，下轮按此继续）
**（可选）需要你拍板：** …（最多一问；多数轮次省略本节）
```

## 流程

### 0. 入场

- 复述用户想法；识别用户是否处于「大量未知」→ 默认 **Default-forward**
- 读 `CONTEXT.md` / ADR / `.scratch`（若有）
- 产物：`.scratch/<feature-slug>/项目建议书.md`（slug 缺省 `product-discovery`）
- **约定议会轮次（必做，仅此一问）：**
  - 问用户：「计划讨论几轮？默认 **10 轮** 后再提议成文。」
  - 用户给数字 → 记为 `目标轮次`；用户说「你定」→ `目标轮次 = 10`
  - 整场议会维护 `当前轮次`（从 1 递增）；每轮 Facilitator 必须写 `第 N / 目标轮次 轮`

### 1. 议会轮次

顺序：**Analyst（按需）→ Advocate → Skeptic → Facilitator**

**成文门槛（硬规则）：**

- `当前轮次 < 目标轮次` 时：**禁止**提议撰写《项目建议书》、禁止问「是否成文」「要不要写文档」。
- 未满轮次时 Facilitator 只许：更新假设登记册、默认推进、（可选）一拍板问题。
- `当前轮次 ≥ 目标轮次` 后，方可**可选**问一句：「是否现在开始写建议书，还是再讨论 N 轮？」——用户未表态则**继续讨论**，勿默认成文。

**提前成文：** 仅当用户**明确**说「可以成文 / 写文档 / 够了」时，可无视轮次下限进入阶段 2。

**禁止早停提议：** 删除「已有推荐路径即可成文」类自动收敛；Skeptic 无新反驳**不能**作为成文理由。

### 2. 成文

按 [PROPOSAL-TEMPLATE.md](./PROPOSAL-TEMPLATE.md) 写入建议书。Analyst 调研摘要并入「问题与机会」「技术选型」；低置信度项进「待验证」。

### 3. 下一步

| 状态 | 建议 |
|------|------|
| 假设仍多、未验证 | 继续本会话或 `/prototype` spike |
| 可写规格 | `/to-prd` → `/to-issues` |
| 要抠术语 | `/grill-with-docs` |

issue 发布须用户同意。

## 规则

- Skeptic 禁止空泛「需要调研」——须给验证实验或「在假设 X 下」的推论
- 技术：默认 **1 推荐 + 1 备选**，非 N 选 1 问卷
- 不写文件路径级实现细节
- 待决 ≠ 已决；猜测标置信度
- **轮次：** 默认至少 **10 轮** 讨论后才可**提议**成文；入场可改目标轮次

## 快速开始

用户：「我想做 XXX，市场和技术都不懂。」

→ 入场约定目标轮次（默认 10）→ 第 1 轮四角色输出（**不成文**）→ … → 第 10 轮后可问是否写建议书 → 用户确认后阶段 2。
